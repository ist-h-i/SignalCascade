## 2. プロジェクト概要

本システムは、価格系列を単なる close-only 時系列として扱うのではなく、**終値を主系列として固定しつつ、OHLC から得られる方向性情報で close 系列を残差的に補正する予測モデル**である。

本システムは以下の5点を中核とする。

1. **Close Anchor**  
   モデルの主軸は常に終値 \(d(i)=C_i\) とし、予測の中心を close に置く。

2. **Path-Averaged Directional Balance の導入**  
   ローソク足内部の上昇・下降バランスを表す補助スカラー \(\chi\) を定義し、close-only モデルの弱点を補う。

3. **加算型 \(L_0\) への置換**  
   旧来の乗算型補正
   \[
   L_0=d_i(1+\cdots)
   \]
   を廃止し、
   \[
   L_0=d_i+\text{残差補正}
   \]
   とすることで、数値安定性・解釈性・単位整合性を確保する。

4. **マルチタイムフレーム主モデル**  
   4時間足・日足・週足を主モデルの時間軸とし、multi-horizon の return / uncertainty / 足形状を同時に推定する。

5. **1時間足 / 30分足の離脱監視オーバーレイ**  
   主シグナルの生成は4時間足系に限定し、短い時間足は 4h レビュー間の逸脱検知と reduce / exit 判断専用に使う。

## 3. 背景と v1.1 からの変更理由

### 3.1 従来の close-only 問題
単純な終値時系列だけでは、同じ close を持つ足同士の違いが失われる。  
例えば以下は close-only では区別が困難である。

- 上髭が長い弱気反転気味の足
- 下髭が長い押し目反発気味の足
- 実体が大きく方向が強い足
- 実体が小さく迷いのある足

### 3.2 旧 \(L_0\) の問題
旧構造では、close に補助項を乗算する形が検討されていたが、以下の問題があった。

- 補正係数が \(-1\) を跨ぐと符号反転し得る
- 補正が過大化すると系列スケールが不安定になる
- 無次元量を価格に直接掛けるため、単位整合性が曖昧になる
- 学習時に補正側が過剰支配しやすい

### 3.3 v1.2 での解決
v1.2 では以下を採用する。

- OHLC 由来の方向バランス指標 \(\chi\) を close-only 改善案として導入
- 補正項を価格単位へ変換するスケール \(a_i\) を導入
- \(L_0\) を加算型残差構造に変更
- 主系列は close に固定しつつ、OHLC 補助情報を残差として取り込む

## 4. システムの目的

本システムの目的は、以下を同時に満たすことである。

1. 次時点または複数 horizon に対する終値予測を行う
2. 次の足形状（上髭・符号付き実体・下髭）を予測する
3. 予測不確実性を別出力として推定する
4. 時間軸ごとに独立した情報をベクトル重みで統合する
5. 主モデルは数時間〜数日保有に適したシグナルを出す
6. 短時間足はリスク管理と早期離脱に限定して使う
7. バックテストおよび将来のリアルタイム推論に耐える因果構造を維持する

## 6. 用語・定義

### 6.1 時間足
- 30m: 30分足
- 1h: 1時間足
- 4h: 4時間足
- 1d: 日足
- 1w: 週足

### 6.2 主モデル
4h / 1d / 1w を使ってエントリー、方向、保有 horizon、目標ポジションサイズを決定するモデル。

### 6.3 離脱オーバーレイ
1h / 30m を使って 4h レビュー間の逸脱を監視し、hold / reduce / full exit / hard exit を決める補助モデル。

### 6.4 Close Anchor
終値を主系列として固定し、その周りに OHLC 由来の補正を加える考え方。

### 6.5 Path-Averaged Directional Balance
足内部の上昇 / 下降バランスを表す正規化スカラー。  
v1.2 では \(\chi\) と記述する。

### 6.6 \(L_5\)
足形状予測ベクトル。  
各 branch または融合後のモデル出力として、
\[
L_5=[\text{上髭}, \text{符号付き実体}, \text{下髭}]
\]
の意味で用いる。

## 7. データ要件

### 7.1 入力データ
最低限必要な入力は以下とする。

- Open
- High
- Low
- Close

Volume が利用可能な場合は追加特徴量として利用する。

### 7.2 元データ粒度
最低でも 30 分足を生成できる粒度の原系列が必要である。  
1h / 4h / 1d / 1w はそこから再集約して生成する。

### 7.3 品質要件
以下の品質要件を満たすこと。

- 欠損値の扱いが明示されていること
- 休日・セッション切れ目の処理が一貫していること
- split / merge / ロールオーバー等の価格補正方針が固定されていること
- 再サンプリング時に future leakage が起きないこと

### 7.4 確定足のみ利用
全ての時間足で、**未確定バーは学習・推論のどちらにも使わない**。

## 8. 派生特徴量要件

### 8.1 終値系列
各時間足 \(\tau\) の終値を

\[
d_i^{(\tau)}=C_i^{(\tau)}
\]

とする。  
本プロジェクトにおける `d(i)` は常に終値である。

### 8.2 足形状自己教師
各足から以下の自己教師ベクトルを作る。

\[
Q_i^{(\tau)}=
\begin{bmatrix}
u_i^{(\tau)}\\
b_i^{(\tau)}\\
\ell_i^{(\tau)}
\end{bmatrix}
\]

ここで

- \(u\): 上髭
- \(b\): 符号付き実体
- \(\ell\): 下髭

である。

### 8.3 方向バランス指標 \(\chi\)
OHLC から方向バランス指標 \(\chi\) を生成すること。  
\(\chi\) は \([-1,1]\) に正規化された値とし、

- \(\chi>0\): 上昇優勢
- \(\chi<0\): 下降優勢
- \(\chi\approx0\): 中立

を表す。

### 8.4 ローカル価格スケール \(a_i\)
補正項を価格単位へ変換するために、ローカルな平均レンジ \(a_i\) を導入すること。

### 8.5 加算型 \(L_0\)
全時間足に対して、以下の思想を満たす \(L_0\) を構成すること。

- close を主系列として保持する
- OHLC 由来情報を価格単位の残差として加える
- 補正量が過大化しないよう制御する

## 9. システム構成要件

### 9.1 主モデル時間軸
主モデルで用いる時間足集合を

\[
\mathcal T_{\text{main}}=\{4h,1d,1w\}
\]

とする。

### 9.2 離脱オーバーレイ時間軸
離脱オーバーレイで用いる時間足集合を

\[
\mathcal T_{\text{ov}}=\{1h,30m\}
\]

とする。

### 9.3 保有 horizon
4時間足基準の horizon 候補を

\[
\mathcal H=\{1,2,3,6,12,18,30\}
\]

とし、4h〜5d 程度を対象とする。

## 11. 非機能要件

### 11.1 因果性
全ての推論は、その時点までに確定した足だけで計算されなければならない。

### 11.2 数値安定性
以下を満たすこと。

- 分母 0 回避のための \(\varepsilon\) を導入する
- \(\chi\) は正規化して \([-1,1]\) に収める
- \(g_i\) は有界関数（例: \(\tanh\)）を用いる
- 補正量は \(a_i\) を介して価格単位へ変換する
- \(L_0\) の補正量に正則化をかける

### 11.3 拡張性
時間足、特徴量、horizon 集合が設定変更で拡張可能であること。

### 11.4 解釈性
少なくとも以下が説明可能であること。

- どの horizon を選んだか
- どの時間足 branch が強く効いたか
- \(L_5\) の意味
- 離脱がなぜ発生したか
- close anchor 補正量がどの程度だったか

### 11.5 再現性
同じデータ・同じ seed・同じ設定で学習した場合、結果が再現可能であること。

## 16. 受入観点

本版では閾値の最終値は実験により確定する。  
ただし、少なくとも以下の観点で受入評価する。

1. 因果性が破れていないこと
2. \(L_0\) 補正量が暴走していないこと
3. shape head の意味が崩れていないこと
4. 検証で no-trade / over-trade のどちらにも極端に偏っていないこと
5. overlay が主モデルのシグナルを過剰に潰していないこと
6. utility 指標がベースラインを上回ること

## 17. 設計上の重要判断

### 17.1 4h を最短主時間軸にする理由
本モデルはスキャルピングではなく、保有期間が数時間〜数日であるため、30m / 1h を主判断軸にするとノイズとマイクロ構造の影響が強すぎる。  
そのため、4h を主判断軸にし、1h / 30m は exit overlay に限定する。

### 17.2 close-only の完全廃止ではなく close anchor 化
close は価格系列の中心的情報であり続けるため、完全に捨てない。  
代わりに OHLC 情報を close の残差補正として取り込む。

### 17.3 外部 Qt 不使用
足形状ベクトル \(Q\) を自己教師として使うことで、人手ラベルへの依存を避ける。

---

## 18. リスクと注意点

- \(\chi\) は新情報というより close-only の補強であり、過信してはならない
- 補正量が強すぎると close anchor の意味が崩れる
- オーバーレイが過敏だと良いポジションを潰す
- 時間足整列を一度でも誤るとリークが生じる
- utility fine-tune は過学習を招きやすいため、上位層中心に限定する

---

## 19. 最終要件まとめ

本システム v1.2 は、以下の要件を満たす統合予測・シミュレーション基盤とする。

- \(d(i)=\) 終値

- Path-Averaged Directional Balance \(\chi\) を導入
- 4h / 1d / 1w 主モデル

- multi-horizon return / uncertainty 予測
- 自己教師 shape 学習
- シミュレーションメソッドと最適化メソッドを標準装備
- purged / embargoed walk-forward 検証
- 数時間〜数日保有を主対象とする
